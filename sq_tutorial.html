sq tutorial for Splunk

inspired by jq
sq is a lightweight JSON filter language implemented as a Splunk streaming command. This page is a hands-on guide with copy/paste SPL examples you can run immediately.

Start here

Run sq after any search that returns events containing JSON in _raw (or another field using field=).

Important: In this tutorial we always use filter="...". It is the most reliable way to pass expressions that contain spaces, parentheses, or pipes.

Quick sanity check

index=your_index sourcetype=your_sourcetype
| head 5
| sq filter=".sha" mode=out out=sha raw=true
| table sha

Test data (single objects and multi objects)

These examples build JSON inline using makeresults so you can test without any external data.

A. multiple events, each event is one JSON object

| makeresults count=3
| streamstats count as n
| eval _raw=case(
    n=1, "{\"id\":1,\"name\":\"Ada\",\"tags\":[\"red\",\"green\",\"blue\"],\"score\":7,\"active\":true}",
    n=2, "{\"id\":2,\"name\":\"Ben\",\"tags\":[\"blue\"],\"score\":null,\"active\":false}",
    1=1, "{\"id\":3,\"name\":\"Cy\",\"tags\":[],\"score\":12,\"active\":true}"
  )
| table _raw

B. a single event, where _raw is a JSON array of objects

| makeresults
| eval _raw="[
  {\"id\":1,\"name\":\"Ada\",\"tags\":[\"red\",\"green\",\"blue\"],\"score\":7},
  {\"id\":2,\"name\":\"Ben\",\"tags\":[\"blue\"],\"score\":null},
  {\"id\":3,\"name\":\"Cy\",\"tags\":[],\"score\":12}
]"
| table _raw
Why two datasets? Most Splunk data arrives as one JSON object per event (A). Sometimes you will ingest one event containing an array of objects (B). Both are valid, and sq works with both.

Core syntax

Field access

| makeresults
| eval _raw="{\"user\":{\"name\":\"Ada\",\"role\":\"admin\"}}"
| sq filter=".user.name" mode=out out=name raw=true
| table name

Array index and iteration

Index (.[0])

| makeresults
| eval _raw="[10,20,30]"
| sq filter=".[0]" mode=out out=first raw=true
| table first

Iterate (.[])

| makeresults
| eval _raw="[10,20,30]"
| sq filter=".[]" mode=emit out=value raw=true
| table value

Pipes

| makeresults
| eval _raw="{\"tags\":[\"red\",\"green\",\"blue\"]}"
| sq filter=".tags[] | ." mode=emit out=tag raw=true
| table tag

Functions

1. select(PRED)

| makeresults count=3
| streamstats count as n
| eval _raw=case(
    n=1, "{\"id\":1,\"name\":\"Ada\",\"score\":7}",
    n=2, "{\"id\":2,\"name\":\"Ben\",\"score\":null}",
    1=1, "{\"id\":3,\"name\":\"Cy\",\"score\":12}"
  )
| sq filter="select(.score != null) | .name" mode=emit out=name raw=true
| table name

2. map(EXPR)

Map expects an array input and returns a new array.

| makeresults
| eval _raw="{\"names\":[{\"name\":\"Ada\"},{\"name\":\"Ben\"},{\"name\":\"Cy\"}]}"
| sq filter=".names | map(.name)" mode=out out=names mv=true raw=true
| table names

3. length

| makeresults
| eval _raw="{\"tags\":[\"red\",\"green\",\"blue\"]}"
| sq filter=".tags | length" mode=out out=tag_count raw=true
| table tag_count

4. keys and has()

keys

| makeresults
| eval _raw="{\"id\":1,\"name\":\"Ada\",\"active\":true}"
| sq filter="keys" mode=out out=keys mv=true raw=true
| table keys

has("key")

| makeresults
| eval _raw="{\"id\":1,\"name\":\"Ada\"}"
| sq filter="has(\"name\")" mode=out out=has_name raw=true
| table has_name

5. Fallback operator //

If the left side is null or false, sq returns the right side.

| makeresults count=2
| streamstats count as n
| eval _raw=case(
    n=1, "{\"name\":\"Ada\",\"nick\":null}",
    1=1, "{\"name\":\"Ben\"}"
  )
| sq filter=".nick // .name" mode=out out=label raw=true
| table label

Output modes and formatting

mode

  • mode=out (default): one output event per input event. Writes into out=.
  • mode=emit: emits one output event per produced value (can increase event count).
  • mode=merge: if the final value is an object, merges its keys into the event.

raw

  • raw=true: for scalars, writes plain values (no JSON quotes).
  • raw=false: non-scalars are JSON encoded, scalars may appear with JSON encoding.

pretty

If pretty=true, objects and arrays are JSON pretty-printed (human readable). If false, JSON is compact (better for storage and further machine processing). 

| makeresults
| eval _raw="{\"a\":1,\"b\":{\"c\":2,\"d\":[3,4]}}"
| sq filter="." mode=out out=obj pretty=true
| table obj

mv

If a filter produces multiple values in mode=out, you can choose how they are stored:

  • mv=false (default): store multiple values as a single JSON array value.
  • mv=true: store multiple values as a Splunk multivalue field (best for later SPL like mvexpand).
| makeresults
| eval _raw="{\"tags\":[\"red\",\"green\",\"blue\"]}"
| sq filter=".tags[]" mode=out out=tags mv=true raw=true
| table tags
| mvexpand tags
Reminder: if you want each produced value as its own event, use mode=emit. If you want one event with a multivalue field, use mode=out mv=true.

Worked examples

Example: filter then return a field

| makeresults count=3
| streamstats count as n
| eval _raw=case(
    n=1, "{\"id\":1,\"name\":\"Ada\",\"active\":true}",
    n=2, "{\"id\":2,\"name\":\"Ben\",\"active\":false}",
    1=1, "{\"id\":3,\"name\":\"Cy\",\"active\":true}"
  )
| sq filter="select(.active == true) | .name" mode=emit out=name raw=true
| table name

Example: nested extraction

| makeresults
| eval _raw="{\"commit\":{\"committer\":{\"name\":\"web-flow\"},\"message\":\"hello\"}}"
| sq filter="{name: .commit.committer.name, message: .commit.message}" mode=out out=commit pretty=true
| table commit

Example: single event with an array of objects

| makeresults
| eval _raw="[
  {\"id\":1,\"name\":\"Ada\",\"score\":7},
  {\"id\":2,\"name\":\"Ben\",\"score\":null},
  {\"id\":3,\"name\":\"Cy\",\"score\":12}
]"
| sq filter=".[] | select(.score != null) | .name" mode=emit out=name raw=true
| table name

Tips and gotchas

Always use filter= in docs: sq filter="select(.score != null) | .name" is more reliable than sq "select(...)" in SPL.

If your JSON is not in _raw, copy it into a field first, then set field=. 

... your search ...
| eval json=_raw
| sq field=json filter=".id" mode=out out=id raw=true
| table id

When experimenting, start with raw=true for simple scalar extractions, and add pretty=true only for readability.
